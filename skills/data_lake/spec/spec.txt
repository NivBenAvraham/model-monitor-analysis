DATA LAKE — Skill Specification
═══════════════════════════════════════════════════════════════════════

PURPOSE
───────
Central connection and table catalog layer for all Athena data lake access.
Every other skill or script that reads from AWS Athena should import from here —
not hardcode connection strings, database names, or table names.

Databases
  data_lake_curated_data  — cleaned, joined, production-ready tables
  data_lake_raw_data      — raw model run logs and device metadata


═══════════════════════════════════════════════════════════════════════
CONNECTION CONFIG
═══════════════════════════════════════════════════════════════════════

  S3_STAGING_DIR   s3://us-east-1-data-analytics-storage/athena-query-results/
  REGION           us-east-1
  WORKGROUP        data_analytics
  CURATED_DATABASE data_lake_curated_data
  RAW_DATABASE     data_lake_raw_data

Credentials expire every 4 hours.
Refresh with:  source scripts/refresh_aws_credentials.sh


═══════════════════════════════════════════════════════════════════════
THREAD SAFETY
═══════════════════════════════════════════════════════════════════════

pyathena connections are NOT thread-safe.

connection.py handles this via threading.local():
  - get_connection(database) returns a per-thread connection
  - Connections are created once per thread and reused
  - Safe to use inside ThreadPoolExecutor workers

For scripts that run once and exit, AthenaSession context manager is
cleaner — opens fresh connections and closes them on __exit__.

TABLES — data_lake_raw_data
═══════════════════════════════════════════════════════════════════════

─────────────────────────────────────────────────────────────────
supervised_beeframes  /  unified_bee_frames  (TABLE SWITCH — IMPORTANT)
─────────────────────────────────────────────────────────────────
Both tables have identical structure.

  date < 2026-03-10  →  supervised_beeframes
  date ≥ 2026-03-10  →  unified_bee_frames

NEVER hardcode either name. Use raw_bee_frames_table(date) from
  skills/temperature_data_export/scripts/queries.py
or
  src/model_monitor/ingestion/__init__.py

Columns:
  sensor_mac_address   string
  pred_raw             double    raw bee_frames prediction
  lambda_name          string    model identifier (full path)
  log_timestamp        timestamp

Dedup pattern (always needed):
  row_number() OVER (
      PARTITION BY sensor_mac_address, date(log_timestamp), lambda_name
      ORDER BY log_timestamp DESC
  ) = 1

─────────────────────────────────────────────────────────────────
gateways
─────────────────────────────────────────────────────────────────
  mac                  string    gateway_mac_address
  hardware_extensions  string    non-empty → hardware extension attached

Join to gateway_samples to derive hardware_extension_present.

FILES
═══════════════════════════════════════════════════════════════════════

  scripts/connection.py   Athena connection factory
                            read_curated(sql)       → DataFrame
                            read_raw(sql)           → DataFrame
                            read(sql, database)     → DataFrame
                            get_connection(db)      → thread-local connection
                            make_reader(db)         → Callable[[str], DataFrame]
                            AthenaSession           → context manager

  scripts/catalog.py      Table name constants + schema reference
                            CURATED.*               table name constants
                            RAW.*                   table name constants
                            SCHEMA                  dict of column → type
                            list_tables(database)   → DataFrame (live)
                            describe_table(t, db)   → DataFrame (live)
                            sample(t, db, n)        → DataFrame (live)
                            schema(table)           → dict (local reference)
